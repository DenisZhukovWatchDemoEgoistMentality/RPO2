# Название БД - Topito_DB
- Использовался: pgAdmin 4

- Database.sql - таблицы базы данных.
- Schema.pgerd и Схема.png - визуальное представление таблиц.

# Структура БД
- Users - таблица пользователей.
- Resourse_Status - enum-таблица статуса объявления.
- Resourses - таблица объявлений.
- Order_Status - enum-таблица статуса заказа.
- Orders - таблица заказов, т.е. если объявление было принято другим пользователем.

# Подробнее об таблицах
- Users:
    1. First_name - Имя, обязательное.
    2. Second_name - Фамилия, необязательное.
- Resourse_Status:
    - Name - Название:
        1. Выставлен.
        2. Продан.
-  Resourses:
    1. Owner_Id - Владелец объявления, обязательное.
    2. Name - Названиеб обязательное.
    3. Description - Описание, необязательное.
    4. Price - Цена, обязательное.
    5. Status - Статус объявления, обязательное, по умолчанию "Выставлен".
- Order_Status:
    - Name - Название:
        1. Ожидание.
        2. Продан.
        3. Отменён.
- Orders:
    1. Vendor_Id - Продавец, обязательное.
    2. Resourse_Id - Объявление, обязательное.
    3. Customer_Id - Покупатель, обязательное.
    4. Status - Статус заказа, обязательное, по умолчанию "Ожидание".
